Glosario de Términos

Internet:

Internet es un conjunto descentralizado de redes de comunicación interconectadas que utilizan la familia de protocolos TCP/IP. Se originó a finales de los años 60 a partir de la red ARPANET. Internet funciona mediante el envio de la información a través de los protocolos TCP/IP.

Protocolos:

Los protocolos de seguridad de red son un tipo de protocolo de red que garantiza la seguridad y la integridad de los datos en tránsito a través de una conexión de red como Internet. Están diseñados principalmente para evitar el acceso no autorizado a los datos de la red. Los protocolos son: HTTP, FTP, SSH y DNS. Los RFC (Request For Comments) son esquemas que representan estándares y su frecuencia de uso acerca de internet.

Niveles ISO-OSI:

El modelo de interconexión de sistemas abiertos o “modelo OSI” es un estándar que tiene por objetivo conseguir interconectar sistemas de procedencia distinta para que estos pudieran intercambiar información sin ningún tipo de impedimentos debido a los protocolos con los que estos operaban de forma propia según su fabricante. Las siguientes capas son:

  • Física
  • Enlace de datos
  • Red
  • Transporte
  • Sesión
  • Presentación
  • Aplicación

Protocolos:

El Protocolo de transferencia de hipertexto es el protocolo de comunicación que permite las transferencias de información en la World Wide Web

HTTP define la sintaxis y la semántica que utilizan los elementos de software de la arquitectura web para comunicarse. HTTP es un protocolo sin estado, es decir, no guarda ninguna información sobre conexiones anteriores. Un paquete HTTP se constituye de una línea, cabeza y cuerpo, mientras que la respuesta se compone del protocolo, un código de estado como 200 OK por ejemplo, cabeza y cuerpo. Las siguientes peticiones de HTTP son:
  • GET: Recuperación de datos
  • HEAD: Similar a GET, pero sin devolver el cuerpo
  • POST: Procesamiento de datos
  • PUT: Similar a POST, pero este no se refiere al dato procesado
  • DELETE: Borrado de datos
  • TRACE: Búsqueda completa de datos. Para depurar
  • OPTIONS: Devuelve los métodos HTTP disponibles
  • CONNECT: Comprobar acceso a host
  • PATCH: Similar a PUT. Usado para actualizar.

Carga de una página web:

Se envía la solicictud al servidor. Este la recibe y comienza el proceso de construir la página web. Si existe código PHP, se ejecuta en este momento. Luego se recibe la página interpretando el HTML. Mientras se interáctua en la página, se procesará el JavaScript. También se puede usar AJAX para hacer peticiones a la BD y puede crear páginas dinámicas.

TCP:

Con el uso de protocolo TCP, las aplicaciones pueden comunicarse en forma segura. Esto significa que los routers sólo tienen que enviar los datos en forma de segmentos, sin preocuparse con el monitoreo de datos porque esta función la cumple el protocolo TCP. Muchos programas dentro de una red de datos compuesta por redes de computadoras, pueden usar TCP para crear “conexiones” entre sí a través de las cuales puede enviarse un flujo de datos. El protocolo garantiza que los datos serán entregados en su destino sin errores y en el mismo orden en que se transmitieron.

  • Permite colocar los segmentos nuevamente en orden cuando vienen del protocolo IP.
  • Permite el monitoreo del flujo de los datos y así evita la saturación de la red.
  • Permite que los datos se formen en segmentos de longitud variada para "entregarlos" al protocolo IP.
  • Permite multiplexar los datos, es decir, que la información que viene de diferentes fuentes en la misma línea pueda circular simultáneamente.
  • Por último, permite comenzar y finalizar la comunicación amablemente.
Los puertos bien conocidos son asignados por la Internet Assigned Numbers Authority (IANA), van del 0 al 1023 y son usados normalmente por el sistema o por procesos con privilegios. Las aplicaciones que usan este tipo de puertos son ejecutadas como servidores y se quedan a la escucha de conexiones. Algunos ejemplos son: FTP (21), SSH (22), Telnet (23), SMTP (25) y HTTP (80). El segmento TCP está compuesto por los datos enviados desde la capa de aplicación y la cabecera añadida por el protocolo de transporte. El segmento TCP es luego encapsulado en un datagrama IP para ser enviado por la capa de red. Socket designa un concepto abstracto por el cual dos procesos (posiblemente situados en computadoras distintas) pueden intercambiar cualquier flujo de datos, generalmente de manera fiable y ordenada. En el TCP/IP, designa los ordenadores de salida y llegada.

IP:

El protocolo de internet (en inglés: Internet Protocol; cuya sigla es IP) es un protocolo de comunicación de datos digitales clasificado funcionalmente en la capa de red según el modelo internacional OSI. Su función principal es el uso bidireccional en origen o destino de comunicación para transmitir datos mediante un protocolo no orientado a conexión que transfiere paquetes conmutados a través de distintas redes físicas previamente enlazadas según la norma OSI de enlace de datos. La dirección IP es un conjunto de números (0 a 255) que identifica, de manera lógica y jerárquica, a una Interfaz en la red de un dispositivo que utilice el protocolo o , que corresponde al nivel de red del modelo TCP/IP El enrutamiento es el mecanismo por el que en una red los paquetes de información se hacen llegar desde su origen a su destino final, siguiendo un camino o ruta a través de la red. En una red grande o en un conjunto de redes interconectadas el camino a seguir hasta llegar al destino final puede suponer transitar por muchos nodos intermedios. El datagrama IP es la unidad de transferencia en las redes IP. Básicamente consiste en una cabecera IP y un campo de datos para protocolos superiores.

URL:

Una URL es un identificador de recursos uniformecuyos recursos referidos pueden cambiar, esto es, la dirección puede apuntar a recursos variables en el tiempo. Una URL se estructura con un protocolo, un subdominio, un dominio, un TLD o extensión de la web, una subcarpeta, una página y ocasionalmente con una etiqueta. os parámetros de URL constan de una clave y un valor separados por un signo igual (=) y unidos por el signo et (&). El primer parámetro siempre aparece después de un signo de interrogación en una URL.

Cliente-Servidor vs P2P:

Mientras que se necesita un servidor fijo en Cliente-Servidor, en P2P cualquiera puede ser servidor y la conexión siempre está activa con un cliente. Asimismo, en Cliente-Servidor, solo el servidor tiene permisos que los clientes no tienen.

Cookies:

Cookie es un término que hace referencia a una pequeña información enviada por un sitio web y almacenada en el navegador del usuario, de manera que el sitio web puede consultar la actividad previa del navegador. Cuando un usuario introduce su nombre de usuario y contraseña, se almacena una cookie para que no tenga que estar introduciéndolas para cada página del servidor. Sin embargo, una cookie no identifica a una persona, sino a una combinación de computadora de la clase de computación-navegador-usuario. Esto significa que la persona que crea la cuenta no se puede presentar con esta cuenta y sea reconocido. Las cookies se suelen almacenar en el lado del cliente en un paquete HTTP.

Tipos MIME:

Un tipo de medio (también conocido como tipo MIME y tipo de contenido) es un identificador de dos partes para formatos de archivo transmitidos por Internet. Sirven para denotar el tipo de los contenidos y adjuntos de los mensajes de Internet

Integración continua:

Consiste en hacer integraciones automáticas de un proyecto lo más a menudo posible para así poder detectar fallos cuanto antes. Se entiende por integración la compilación y ejecución de pruebas de todo un proyecto. Suelen utilizarse aplicaciones como Solano CI, Bamboo, Pipeline o Apache Continuum.
Una prueba unitaria es una forma de comprobar el correcto funcionamiento de una unidad de código. Se suelen usar herramientas como JUnit o Typemock. Las pruebas Mock son pruebas unitarias que utilizan objetos simulados ("mock") que sustituyen a los objetos reales utilizados por la clase o fragmento de código a probar.
Pruebas integrales son aquellas que se realizan una vez que se han aprobado las pruebas unitarias y lo que prueban es que todos los elementos unitarios funcionen. Algunas herramientas son Postman, Github, Notepad++ o FileZilla.

CDN:

Una red de distribución de contenidos (CDN) es una red superpuesta de computadoras que contienen copias de datos, colocados en varios puntos de una red con el fin de maximizar el ancho de banda para el acceso a los datos de clientes por la red. Estática vs Dinámica:

Una Web estática es aquella página enfocada principalmente a mostrar una información permanente, donde el navegante se limita a obtener dicha información, sin poder interactuar con la página web visitada, mientras que una Web dinámica contiene aplicaciones para que el usuario interáctue con ellas.

WebSocket:

WebSocket es una tecnología que proporciona un canal de comunicación bidireccional y full-duplex sobre un único socket TCP. Está diseñada para ser implementada en navegadores y servidores web, pero puede utilizarse por cualquier aplicación cliente/servidor.

Patrones de diseño:

Los patrones de diseño son unas técnicas para resolver problemas comunes en el desarrollo de software y otros ámbitos referentes al diseño de interacción o interfaces. Los siguientes son:

  • MVC: Es un patrón de arquitectura de software que separa los datos y la lógica de negocio de una aplicación de la interfaz de usuario y el módulo encargado de gestionar los eventos y las comunicaciones
  • Singleton: Garantiza la existencia de una única instancia para una clase y la creación de un mecanismo de acceso global a dicha instancia.
  • Factory: Centraliza en una clase constructora la creación de objetos de un subtipo de un tipo determinado.

Callback:

Es una función "A" que se usa como argumento de otra función "B". Cuando se llama a "B", ésta ejecuta "A".

CORS:

El intercambio de recursos de origen cruzado o CORS es un mecanismo que permite que se puedan solicitar recursos restringidos en una página web desde un dominio diferente del dominio que sirvió el primer recurso.

ECMAScript:

ECMAScript define un lenguaje de tipos dinámicos ligeramente inspirado en Java y otros lenguajes del estilo de C. Soporta algunas características de la programación orientada a objetos mediante objetos basados en prototipos y pseudoclases.

Polyfill:

En el desarrollo web, un polyfill es un código que implementa una característica en los navegadores web que no la soportan.

Procesamiento asíncrono:

La programación asíncrona facilita que se pueden ejecutar varios procesos al mismo tiempo dando la sensación de ser más rápido. El callback hell se produce cuando hay varios callbacks en espera y no llegan a completarse. En la informática, la programación reactiva es un paradigma de programación declarativa que se ocupa de los flujos de datos y la propagación del cambio.

Programación reactiva funcional:

La programación reactiva funcional es un paradigma de programación para la programación reactiva que utiliza los elementos básicos de la programación funcional. ReactiveX es un paradigma integral multiplataforma para implementar una programación reactiva funcional tratando los datos como flujos de observables.

Microservicios:

La arquitectura de microservicios es una aproximación para el desarrollo de software que consiste en construir una aplicación como un conjunto de pequeños servicios, los cuales se ejecutan en su propio proceso y se comunican con mecanismos ligeros.

Un CMS te permite mantener el contenido de tu sitio web por ti mismo. Por lo general, a través de una interfaz web simple, el CMS te permite añadir, actualizar y eliminar los textos e imágenes de tu webo de forma sencilla y segura. Las herramientas de CRM son utilizadas por los equipos de atención al cliente, marketing y ventas para realizar un seguimiento de los clientes, propuestas de venta y clientes potenciales.

Programación imperativa:

La programación estructurada es un paradigma de programación orientado a mejorar la claridad, calidad y tiempo de desarrollo de un programa de computadora recurriendo únicamente a subrutinas y tres estructuras básicas. La programación modular es un paradigma de programación que consiste en dividir un programa en módulos o subprogramas con el fin de hacerlo más legible y manejable.

Programación Orientada a Objetos:

SOLID son cinco principios para la POO: Responsabilidad única, abierto/cerrado, sustitución, segregación de interfaces e inversión de la depedencia. Una clase es una especie de "plantilla" en la que se definen los atributos y métodos predeterminados de un tipo de objeto. En programación modular, y más específicamente en programación orientada a objetos, se denomina encapsulamiento al ocultamiento del estado, es decir, de los datos miembro de un objeto de manera que solo se pueda cambiar mediante las operaciones definidas para ese objeto. Un objeto es una instancia de una clase. La herencia es la relación entre una clase general y otra clase más específica. El polimorfismo se refiere a la propiedad por la que es posible enviar mensajes sintácticamente iguales a objetos de tipos distintos.

Programación Orientada a Objetos en Javascript:

Las clases se construyen con funciones que funcionan cmo constructor a la vez.

Interfaz fluida:

Una interfaz fluida es una construcción orientada a objeto que define un comportamiento capaz de retransmitir el contexto de la instrucción de una llamada subsecuente.

Inyección de dependencias e inversión de control:

En informática, inyección de dependencias es un patrón de diseño orientado a objetos, en el que se suministran objetos a una clase en lugar de ser la propia clase la que cree dichos objetos y se encarga de extraer la responsabilidad de la creación de instancias de un componente para delegarla en otro. Inversión de control es un principio de diseño de software en el que el flujo de ejecución de un programa se invierte respecto a los métodos de programación tradicionales.

Programación Orientada a Aspectos:

La Programación Orientada a Aspectos o POA es un paradigma de programación que permite una adecuada modularización de las aplicaciones y posibilita una mejor separación de responsabilidades. Gracias a la POA se pueden encapsular los diferentes conceptos que componen una aplicación en entidades bien definidas, eliminando las dependencias entre cada uno de los módulos.

Excepciones:

El manejo de excepciones es una técnica de programación que permite al programador controlar los errores ocasionados durante la ejecución de un programa informático. Existen herramientas que ayudan a manejarlas tal es el caso de los bloques try que encierran el código que puede lanzar una excepción y los bloques catch que lidian con las excepciones que surjan.

Promesa:

En Programación, un valor futuro es un remplazo para un resultado que todavía no está disponible, generalmente debido a que su cómputo todavía no ha terminado, o su transferencia por la red no se ha completado.

Callback Hell:

El callback hell es la mala utilización de los callbacks, anidando funciones y esperando que funcione de una forma linear.

Métodos async/await:

async-await permite la ejecución de código asíncrono. Ayuda a mantener su interfaz de usuario receptiva en todo momento, mientras ejecuta operaciones potencialmente largas en segundo plano.

Genericidad:

La genericidad es un mecanismo que permite escribir trozos decódigo genérico, que incluye referencias a uno varios nombres de tipos de datos o clases sin que exista una declaración de los mismos. Sobre la regla de + genérico = + reusable && más díficil de mantener porque el código es menos concreto y no se tienen soluciones específicas en caso de error.

Lenguaje de programación:

Un lenguaje de programación es un lenguaje formal que le proporciona a una persona, en este caso el programador, la capacidad de escribir una serie de instrucciones o secuencias de órdenes. Uno compilado depende de un compilador mientras que uno interpretado se basan en instrucciones libres.

Compilador, intérprete y transpilador:

Compilador, que analiza el programa fuente y lo traduce a otro equivalente escrito en otro lenguaje. Intérprete, que analiza el programa fuente y lo ejecuta directamente, sin generar ningún código equivalente. Transpilador es un tipo especial de compilador que traduce de un lenguaje fuente a otro fuente también de un nivel de abstracción parecido. Se diferencia de los compiladores tradicionales en que estos reciben como entrada ficheros conteniendo código fuente y generan código máquina del más bajo nivel.

Fuertemente tipado vs débilmente tipado:

Un lenguaje fuertemente tipado tiene reglas de tipeo más estrictas en el momento de la compilación, lo que implica que es más probable que ocurran errores y excepciones durante la compilación. Por otro lado, un lenguaje débilmente escrito tiene reglas de escritura más flexibles y puede producir resultados impredecibles o puede realizar una conversión de tipo implícita en tiempo de ejecución. Un lenguaje débilmente tipeado realizará algunas coacciones de tipo mientras que uno fuertemente tipeado requerirá que los tipos no coincidentes sean coaccionados o convertidos

Time to market:

Es el tiempo desde la concepción de una aplicación hasta su lanzamiento.

Sobrecarga de Operadores:

Sobrecarga es la capacidad de un lenguaje de programación, que permite nombrar con el mismo identificador diferentes variables u operaciones. En programación orientada a objetos la sobrecarga se refiere a la posibilidad de tener dos o más funciones con el mismo nombre pero funcionalidad diferente.

Interpolación de cadenas:

La característica de interpolación de cadenas se basa en la característica de formato compuesto y proporciona una sintaxis más legible y cómoda para incluir resultados de expresiones con formato en una cadena de resultado.

Bower mantiene actualizadas las librerías o frameworks. Npm es el que va por defecto para node.js y Yarn es un reciente instalador creado por Facebook que otorga mayor gestión al programador.

Yeoman:

Yeoman se ejecuta como una interfaz de línea de comandos escrita para Node.js y combina varias funciones en un solo lugar, como la generación de una plantilla de inicio, la gestión de las dependencias, la ejecución de pruebas de unidad, el suministro de un servidor de desarrollo local y la optimización del código de producción para el despliegue.

Babel:

Babel es un transcompilador que se usa para convertir código ECMAScript6+ a versiones anteriores de Javascript y que los navegadores la puedan procesar.

Paquetes de módulos:

Webpack es un paquete de módulos principalmente para Javascript. Webpack genera un gráfico de depedencias que permite que los desarrolladores lo usen de forma modular. Browserify permite a los desarrolladores escribir en Node.js para navegadores. Parcel ofrece gran velocidad con una mínima configuración.

UglifyJS:

UglifyJS es un parseador minimalista para Javascript.

Minify:

Minify es un minimizador para Javascript.

Aplicación isomórfica:

Una aplicación isomórfica es aquella que su código funciona tanto para cliente o con servidor de Node.js.

DOM y BOM:

DOM ('Modelo de Objetos del Documento') es una interfaz de plataforma que proporciona un conjunto estándar de objetos para representar documentos HTML, XHTML y XML. Y el BOM se enfoca en el navegador en todas sus áreas.

Aplicación web progresiva:

Una aplicación web progresiva es aquella entregada a través de la web y que funciona similar a una aplicación web. Incluye ventajas como trabajar sin conexión y acceder al hardware del dispositivo. Útil para móviles.

Meteor:

Meteor es un framework creado con Javascript diseñado para la creación de prototipos y código multiplataforma.

JHipster:

JHipster es un generador de aplicaciones web y microservicios que usa Angular, Reacy y el framework Spring.

Javascript no obtructivo:

JavaScript no obstructivo incluye la separación de la funcionalidad JavaScript de las capas de estructura/contenido y de presentación de una página y el uso de buenas prácticas a fin de evitar los problemas de incompatibilidad de la programación tradicional en JavaScript.

Array Javascript:

Los arrays en javascript son variables en longitud como tipo.

Objeto Javascript:

En JavaScript, un objeto es un entidad independiente con propiedades y tipos.

Funciones Javascript:

En JavaScript, una función es un conjunto de instrucciones con entrada y salida que deben ser definidas.

Prototipo y Herencia Javascript:

En JavaScript, un prototipo es una propiedad privada que enlaza a otro objeto. En las herencias el this apunta al heredado y no al original.

Cierres Javascript:

Un closure es una función para el entorno de otra función.

Objetos browser y window Javascript:

El objeto browser representa el BOM mientras que el objeto window representa el DOM, ambas proporcionando referencias a cada modelo respectivamente.

Objetos Wrappers:

Math es un objeto totalmente estático y sirve para funciones matemáticas Date permtie insertar una fecha y hora. El objeto RegExp permite crear un objeto que contenga una expresión regular. El objeto JSON analiza y convierte a JSON. No puede ser construido ni llamado.

Reveal.js permite crear presentacioens en la página web de forma gratuita.

BD vs SGBD vs SGBDR:

Una base de datos es un conjunto de datos no redundantes y un sistema de gestion de base de datos es el conjunto de programas que permiten definir, manipular y utilizar la información que contienen las bases de datos y una SGBDR es la que sigue el modelo relacional.

SGBDs libres y relacionales:

Los sistemas importantes son MYSQL, ProsgretSQL, Oracle. Los lbires son aquellos gratuitos. y los SGBDs relacionales son los que siguen el sistema relacional.

  • Redis: Redis está pensado para guardar claves en hashes, pero tambien puede usarse como una BD.

Campos autonuméricos y claves:

Campo cuyo valor va aumentando secuencialmente con un incremento de 1 o a partir de una secuencia aleatoria de números. La clave es aquella que identifica de manera única a un campo y las ajenas aquellas que se comparten entre campos relacionados.

Vista:

Una vista es el conjunto de resultados de una consulta almacenada en los datos.

Diagrama Entidad-relación:

Normalización:

La normalización de bases de datos es un proceso que consiste en designar y aplicar una serie de reglas a las relaciones obtenidas tras el paso del modelo entidad-relación al modelo relacional. Las siguientes son:

  • Primera: Atributos únicos, número igual de columnas, dependencia funcional, indepedencia fila-columna
  • Segunda: Primera + Ausencia de dependencias parciales
  • Tercera: Segunda + Ausencia de depedencias transitivas

DML y DDL:

Es un lenguaje de programación para definir estructuras de datos, proporcionado por los sistemas gestores de bases de datos, en este caso PostgreSQL. En inglés, Data Definition Language, de ahí sus siglas DDL. En inglés, Data Manipulation Language (DML). Utilizando instrucciones de SQL, permite a los usuarios introducir datos para posteriormente realizar tareas de consultas o modificación de los datos que contienen las Bases de Datos.

Lenguaje SQL:

  • Inserciones: Añade registros a una tabla
  • Borrados: Borra registros a una tabla
  • Actualizaciones: Actualiza registros a una tabla
  • Consultas: Permite ver una o varias tablas
  • Alias: Permite dar un nombre distintivo que puede ayudar a realizar consultas.
  • Where: Condiciona las consultas
  • Consultas agrupadas: Agruapa las consultas segun un atributo.
  • Consultas agrupadas y filtradas: Idem que la de arriba pero con índices filtrados.

ResultSet:

Un ResultSet contiene los resultados de una consulta SQL, por lo que es un conjunto de filas obtenidas desde una base de datos.

Transacción:

Una transacción es una interacción con una estructura de datos compleja, compuesta por varios procesos que se han de aplicar uno después del otro. La transacción debe realizarse de una sola vez y sin que la estructura a medio manipular pueda ser alcanzada por el resto del sistema hasta que se hayan finalizado todos sus procesos.

Prepared statement e inyección de código:

Prepared statement ejecuta un statement común de la BD con gran eficiencia todo repetidas veces. Un defecto de inyección de código ocurre cuando es posible enviar datos inesperados a un intérprete.

  • TOAD: Es exclusiva de Microsoft

Control de versiones:

Se llama control de versiones a la gestión de los diversos cambios que se realizan sobre los elementos de algún producto o una configuración del mismo

Ventajas que aportan los controladores de versiones:

  • Trazabilidad: Debe saberse lo que hace cada cambio.
  • Historial de cambios: Poder consultar todo tipo de cambios realizados por todos.
  • Reversión: Revertir dichos cambios en caso de que no sean de nuestro agrado.
  • Trabajo en grupo: Permite una mayor coordinación entre grandes grupos.
  • Trabajo en ramas: permite mantener tantos flujos de desarrollo como sean necesarios.

Herramientas:

  • Cvs: Implementa control de versiones, mantiene registros y cambios en un archivo. Descontinuado desde 2008.
  • Svn: Implementa control de versiones y solo guarda modificaciones. Funciona entre redes.
  • Git: Implementa control de versiones diseñado para grandes proyectos.
  • Mercurial: Implementa control de versiones y ofrece gran rendimiento y gestión de archivos pesados.

Repositorios git:

  • Github: Github es una plataforma para alojar proyectos con la herramienta git.
  • Bitbucket: Bitbucket es un servicio de alojamiento para los proyectos que utilizan el sistema de control de versiones Mercurial y Git.
  • Gitlab: Gitlab es un servicio web de control de versiones que ofrece también alojamiento de wikis y un sistema de seguimiento de errores.

  • Phpunit: Phpunit permite crear pruebas unitarias para PHP.

  • TeamCity: Es un servidor de integración compatible con la mayoría de sistemas de versión de control.
  • CircleCI: Inspecciona archivos y encuentra problemas usando SSH.

  • SQS: SQS es un servicio de distribución de colas de mensajes. Soporta el envío programado de mensajes a través de aplicaciones de servicios web como una forma de comunicación a través de Internet.
