Términos del Glosario

HTTP y Redes

Internet: es una abreviación de un anglicismo que significa Red Internacional de Computadores. Se creó en la década de los 60, de la mano de ARPA (DARPA actualmente) por la necesidad de dar un mejor uso a los ordenadores de aquel entonces y por hacer más fácil la comunicación entre estos dispositivos conectándolos en red. Además, su funcionamiento se basa en ofrecer servicios para los usuarios que otros usuarios han creado, ya sea a través de hipertextos, correo electrónico, archivos multimedia, sistemas de telefonía, televisión, etc.

Protocolo: es un sistema de normas que regulan la comunicación entre uno o más sistemas. Hay distintos tipos de protocolo, y pueden ser: TCP/IP, HTTP, FTP, SSH, DNS...

Modelo ISO/OSI: es el modelo de interconexión de sistemas abiertos. Lo creó la Organización Internacional de la Normalización y su objetivo es permitir la comunicación sin impedimentos entre ordenadores de distinta procedencia consiguiendo hacer un modelo estándar en la funcionalidad del protocolo sea cual sea el que se use.

Protocolo HTTP: es el protocolo de comunicación que permite la transferencia de información en la World Wide Web. Es un protocolo sin estado, es decir, no guarda información sobre conexiones anteriores. Por ello, en el desarrollo web es necesario el uso de la cookies.

Cuando trabajamos con Web APIs, la comunicación se realiza a través del protocolo HTTP, haciendo el cliente una petición a la API y ésta enviando una respuesta.
La estructura de la petición se divide en:

  • Línea de petición
  • Cabecera
  • Cuerpo
La estructura de la respuesta se divide en:
  • Línea de estatus
  • Cabecera
  • Cuerpo

Por otro lado, los principales métodos HTTP son:

  • GET: se utiliza para adquirir un recurso que se encuentra en el servidor web.
  • POST: se utiliza para para enviar información a un recurso web del servidor.
  • PUT: se utiliza para crear/cargar un recurso nuevo en el servidor. Si éste ya existe, lo reemplaza.
  • DELETE: se utiliza para borrar un recurso en específico del servidor web

Proceso de carga de una página web: para empezar, el cliente envía una solicitud al servidor. Éste la recibe y comienza a construir tu página web. En ese momento, el código del servidor se ejecuta y se hacen todas las consultas a la base de datos. Como resultado, se obtine una página con HTML y JavaScript.
Después, el navegador recibe la página y la interpreta, aquí aparece en tu pantalla. Mientras estás en ella, el JavaScript interactuará contigo respondiendo a los eventos, y gracias a AJAX, este recibirá la información y cambiará elementos en la interfaz sin necesidad de hacer múltiples peticiones al servidor.

TCP: es uno de los protocolos fundamentales de Internet. Se utiliza para crear conexiones entre ordenadores donde se transmiten datos y se reciben sin errores y en el orden que se transmitieron.
Los puertos bien conocidos de TCP son: FTP(21), SSH(22), Telnet(23), SMTP(25) y HTTP(80). Son utilizados normalmente por procesos con privilegios o aplicaciones que se usan como servidores.
Los segmentos TCP están formados por los datos enviados desde la capa de la aplicación y por la cabecera añadida por el protocolo de transporte.
Un socket designa un concepto abstracto por el cual dos procesos en ordenadores distinos pueden intercambiar flujo de datos de forma ordenada y fiable.

IP: es un protocolo de comunicación de datos digitales que sigue el modelo OSI. Su función principal es el uso bidireccional de la comunicación para transmitir datos mediante un protocolo no orientado a conexión.
Una dirección IP es un conjunto de números que identifica a una Interfaz en la red de un dispositivo que usa IP o TCP/IP.
El enrutamiento es el mecanismo por el que un paquete llega desde su origen a su destino final en una red.
El datagrama IP es la unidad de transferencia en las redes IP. Su estructura consiste en una cabecera IP y un campo de datos para protocolos superiores.

URL: es una cadena de texto que sirve para identificar un recurso de red en Internet. Su estructura se divide en tres partes:

  • La designación del protocolo
  • El nombre de host o dirección
  • La ubicación del archivo o recurso

Aplicaciones cliente-servidor vs P2P: la diferencia entre estos tipos es que, en las cliente-servidor, existe un cliente que únicamente se encarga de hacer peticiones a un servidor, que se encarga de enviar las respuestas. Por otro lado, en las P2P, los ordenadores hacen de cliente y servidor al mismo tiempo.

Cookie: es un archivo de pequeño tamaño enviado por un sitio web y almacenado por el navegador del usuario. Sirven, generalmente, para identificar al usuario que visita la web y su actividad en la misma.
Como la cookie registra la actividad del usuario en la web, se puede establecer una sesión de esa manera, gracias a que la cookie almacena la actividad del usuario y recuerda lo que éste hace, de manera que no hace falta pedir información al usuario constantemente ni que se identifique.
La cookie viaja en el paquete HTTP como parámetro del mismo.

MIME: es una forma estandarizada de indicar la naturaleza y el formato de un documento, archivo o conjunto de datos. Se utilizan por los navegadores para determinar como se procesará un documento.

Proyectos Web

  • Requerimientos y especificaciones de una aplicación: son los estándares que necesita cada usuario para poder ejecutar la aplicación web.
  • Metodologías del desarrollo del software: +info
    • Modelo en cascada: método secuencial en el que los pasos del desarrollo son vistos hacia abajo. Se mantiene un control muy estricto de la aplicación gracias a una amplia documentación.
    • Modelo en espiral: método que se centra en la evaluación y reducción de riesgo del proyecto diviendolo en segmentos más pequeños y proporcionar más facilidad de cambio durante el proceso de desarrollo.
    • Modelos basados en prototipos: método de desarrollo en el cual se desarrolla un modelo básico con las funcionalidades mínimas de la aplicación. Así, el cliente puede probar el proyecto y comunicarse con los desarrolladores para saber si se está cumpliendo con las espectativas y funcionalidades acordadas.
    • Metodologías ágiles:
      • Métodos Kanban: es un sistema basado en la señalización, en el que se visualizan las tareas de producción por demanda a través de tarjetas. Es decir, el proceso superior se ejecuta si el inferior realmente lo necesita, evitando así la sobreproducción.
      • Extreme programming: es una metodología que se centra en la adaptabilidad en vez de en la previsibilidad. Los cambios de requisitos se hacen sobre la marcha y hay que adaptarse independientemente del momento de vida en el que esté el proyecto. +info
      • Scrum: es una metodología que reune un conjunto de buenas prácticas para trabajar en equipo y obtener el mejor resultado posible en un proyecto. Es muy indicado para proyectos en entornos complejos, donde se necesitan resultados pronto y la innovación, competitividad, flexibilidad y productividad son fundamentales. +info
  • Integración continua:
    • Pruebas unitarias: forma de comprobar el correcto funcionamiento de una unidad de código.
    • Pruebas de integración: tras realizar las pruebas unitarias, se hacen las pruebas de integración. Estas son en las que se unen todos los elementos unitarios para comprobar si juntos funcionan correctamente.
    • TDD (Test-Driven Development): es una práctica que consiste en escribir las pruebas unitarias, después el código fuente que pase esas pruebas, y por último, refactorizar el código.
  • Documentación de programas: es el conjunto de documentos que se entrega con un determinado software y que contiene las instrucciones para que el usuario sepa como funciona y aprenda a manejarlo. Hay dos tipos: interna y externa.
    • Documentación interna: es la que se encuentra dentro del código en forma de comentarios o archivos dentro de la aplicación.
    • Documentación externa: es la que se escribe, por ejemplo, en libros y es ajena a la aplicación en sí.
  • Contenedores: entornos ligeros que proporcionan a la aplicación los archivos, variables y bibliotecas que necesitan para ejecutarse.
  • Gestores de versiones: son los programas que facilitan la administración de las distintas versiones de cada producto desarrollado.
  • Repositorio de código: lugar donde el código está almacenado y desde donde se puede distribuir.
  • Evaluación de la calidad del software: proceso que debe hacerse cada cierto tiempo a un software para asegurar la calidad que éste tiene. Se evalúa principalmente la calidad de los procesos a seguir para su desarrollo y si cumple la normativa.

Diseño de Aplicaciones Web

  • API de servidor: significa interfaz de programación de aplicaciones y es un conjunto de definiciones y protocolos que se utilizan para integrar y desarrollar el software de las aplicaciones. Permite que se comuniquen sus productos y servicios con otros sin necesidad de saber como están implementados.
  • Diagrama entidad-relación: diagrama que muestra como las entidades se relacionan entre sí en una base de datos.
  • Normalización:
    • 1ª forma: todos los atributos son átomicos, la tabla contiene una clave primaria única y no puede ser nula, no debe existir variación en el número de columnas, los campos no clave deben identificarse por la clave, debe existir independencia en el orden tanto de filas como de columnas.
    • 2ª forma: se cumple lo mismo que en la 1ª además de que los atributos no clave dependen de la clave principal.
    • 3ª forma: se cumple lo mismo que en la 2ª además de que no debe existir ninguna dependencia transitiva entre los atributos que no son clave.
  • Restricciones de integridad: son las restricciones que se encargan de asegurar que las modificaciones hechas a la base de datos por usuarios autorizados no provoquen una pérdida en la consistencia de los datos.
  • BD vs SGBD - SGBDR: en una base de datos, los datos son independientes de la aplicación, por tanto, pueden ser usados por cualquiera de ellas y se visualiza normalmente como una hoja de cálculo. En un SGBD, se ha de configurar qué tipo de datos hay, la estructura, las restricciones y controlar la concurrencia, además de que esos datos pueden manipularse con consultas, actualizaciones... Por último, los SGBDR son muy parecidos a los SGBD, pero éstos últimos son para bases de datos relacionales.
  • DML - DDL: SQL las sentencias DDL son para crear bases de datos y sus componentes. Las sentencias DML son para insertar, modificar, borrar y consultar datos en la base de datos.
  • noSQL: sistema de gestión de base de datos que difiere del clásico SGBDR y no utiliza solo lenguaje SQL, aunque sí lo soporta (Not Only SQL = NoSQL). Se caracteriza por no requerir de estructuras fijas para almacenar la información y por escalar bien horizontalmente. +info
  • UML: diagrama de procesos: es la representación gráfica de un algoritmo o procesos de una aplicación.
  • Mockups de cliente: diseño digital de una web o aplicación. Está diseñado, en ocasiones, por aplicaciones de edición de imágenes o con herramientas especiales de maquetas. +info
  • Wireframers: forma básica de plantilla que no incluye color, tipografías, imágenes o gráficos. Se consideran diseños de baja fidelidad.
  • Prototipos: los prototipos son, a diferencia de las maquetas clásicas estáticas, siempre interactivos, y contienen la mayoría de las funciones de una página web o aplicación planificada.
  • Diagrama de navegación de un sitio web: representación gráfica de la organización de la información de una estructura web. +info

Desarrollo de Aplicaciones Web

  • Lenguajes de marcas:
    • HTML: lenguaje de marcas para elaborar páginas web.
    • CSS: lenguaje para definir el diseño de un lenguaje de marcas.
      • Frameworks:
        • Bootstrap: framework de código abierto para diseños de sitios o aplicaciones web.
        • Materialize: framework que facilita el diseño web para un escritorio o dispositivos móviles de cualquier tamaño. +info
        • Foundation: framework que proporciona una cuadrícula responsive e incluye componentes de interfaz de HTML y CSS, plantillas, código... +info
        • Bulma: framework de CSS de código abierto basado en Flexbox y hecho con Sass.
        • Tailwind: framework que contiene un marco de CSS para crear interfaces personalizadas. No tiene un tema predeterminado ni componentes UI. +info
        • Semantic UI: framework para crear el diseño de interfaces web utilizando HTML y CSS legibles. +info
        • UIkit: framework para diseñar páginas web que incluye archivos CSS para normalizar el estilo de los elementos, CSS para impresión y plantilla grid para hacer páginas web responsivas. +info
      • Precompiladores:
        • Sass: precompilador CSS que genera hojas de estilo añadiéndoles de forma automática características que no tiene CSS como variables, funciones, herencia, etc. +info
        • Less: precompilador CSS que dota a éste de comportamiento dinámico.
    • XML: lenguaje de marcas que define un conjunto de reglas para la codificación de documentos.
    • JSON: formato de texto sencillo para el intercambio de datos.
  • Expresiones regulares: secuencia de caracteres que conforman un patrón de búsqueda.
  • CDN: (Content Delivery Network) es una red de entrega de contenidos compuesta por diferentes ordenadores. +info
  • AJAX: es una técnica de desarrollo web para crear aplicaciones interactivas. Se ejecutan en el cliente pero tienen una comunicación asíncrona con el servidor.
  • Páginas estáticas vs páginas dinámicas: las páginas web estáticas únicamente utilizan lenguajes de marcas como HTML y hojas de estilos como CSS, de manera que no tienen ningún tipo de funciones y no varían. A diferencia de las dinámicas, éstas últimas sí tienen funciones y variables, además de otros lenguajes, como JavaScript, que le da dinamismo y permite al usuario interactuar con los componentes de la página.
  • Tecnologías websockets: tecnología que proporciona un canal de comunicación bidireccional y full-duplex sobre un único socket TCP. Se puede implementar en navegadores, servicios web y aplicaciones cliente/servidor. +info
  • Callback: es una función que se usa como argumento para otra función. Cuando se llama a la segunda, la primera se ejecuta. +info
  • CORS: mecanismo que utiliza cabeceras HTTP y permite que un cliente obtenga permisos para acceder a recursos seleccionados desde un servidor en un origen distinto al que pertenece. +info
  • Ecmascript: estándar que actualmente se encarga de regir como debe ser interpretado y debe funcionar JavaScript.
  • Polyfill: fragmento de código que se utiliza para implementar una funcionalidad moderna en navegadores antiguos que no la admiten de forma nativa. +info
  • Procesamiento asíncrono: proceso que ejecuta una tarea en segundo plano, es decir, el usuario no debe esperar a que se termine de ejecutar una tarea.
  • Programación funcional reactiva: paradigma enfocado en el trabajo con una cantidad de datos finita o infinita de forma asíncrona. +info
  • Microservicios: servicios independientes en una aplicación que se despliegan según se vayan necesitando.
  • Diferencias entre CMS, ERP y CRM: son tres gestores de contenido que se diferencian de la siguiente manera. CMS gestiona el contenido y la estructura de un sitio web, lo que ve el público. CRM gestiona los contactos. Y ERP gestiona el inventario, facturas, precios, etc.

Programación y organización del código

  • Código spaguetti: nombre despectivo que se atribuye a los programas cuyo código o estructura es incomprensible.
  • Abstracción: término que hace referencia a elementos del código aislado en los que importa más las entradas que recibe o respuestas que produce que cómo funciona.
  • Paradigma declarativo: es un paradigma basado en el desarrollo de programas describiendo el problema que se quiere solucionar y lo que se quiere resolver a nivel de usuario pero no las instrucciones necesarias para hacerlo. +info
  • Paradigma imperativo: es un paradigma basado en el desarrollo de programas que se describe paso a paso un conjunto de instrucciones que deben ejecutarse para variar el estado del programa y hallar la solución. +info
  • Programación funcional: es un paradigma de programación declarativa basado en el uso de verdaderas funciones matemáticas. +info
  • Programación estructurada: es un paradigma de programación orientado a mejorar la calidad, claridad y tiempo de desarrollo recurriendo únicamente a subprogramas y tres estructuras básicas: secuencia, selección e iteración. +info
  • Programación modular: es un paradigma de programación que consiste en dividir un programa en subprogramas para hacerlo más legible y manejable. +info
  • Programación orientada a objetos:
    • Principios SOLID: son los 5 principios para poder desarrollar un software de calidad. Implican un software eficaz, robusto y estable, que el código sea limpio, flexible y mantenible y que permita escalabilidad.
    • Interfaces: medios comunes para que los objetos no relacionados se comuniquen entre sí.
    • Clases: plantilla en la que se define los atributos y métodos de un tipo de objeto.
    • Encapsulación: ocultamiento de los datos miembro de un objeto de manera que solo pueden alterarse mediante las operaciones definidas para ese objeto.
    • Objetos: entidades provistas de un conjunto de atributos y métodos.
    • Herencia: mecanismo que se utiliza para crear nuevas clases a partir de otras existentes, y que las subclases heredarán el comportamiento y los atributos de la superclase.
    • Polimorfismo
      • basado en herencia: consiste en crear una concordancia en los tipos de datos basada en que una clase es raíz de la otra.
      • basado en implementación de interfaces: consiste en la creación de una interfaz que proporcione un método que puede ser utilizado por distintas clases.
  • Particularidades de la orientación a objetos en JavaScript: principalmente, JS se basa en la programación de funciones, pero a largo plazo, crea un alto nivel de riesgo y muchos fallos, además de dar paso a un código espagueti. Sin embargo, con la orientación a objetos, el código es más fácil de comprender y rastrear mentalmente.
  • Interfaces fluidas: herramienta utilizada para eliminar código innecesario para realizar tareas frecuentes sustituyéndolo por un encadenamiento de métodos, natural y fluido.
  • Inyección de dependencias: patrón de diseño orientado a objetos que da objetos a una clase que necesita para poder funcionar correctamente.
  • Programación orientada a aspectos: paradigma de programación que se basa en tratar las obligaciones transversales de nuestro programa como aspectos (módulos separados) para lograr una correcta separación de responsabilidades. +info
  • Utilidad de las excepciones y sus tratamientos: las excepciones son indicaciones de problemas que surgen durante la ejecución de un programa. Su manejo es muy importante ya que permite al usuario seguir utilizando la aplicación sin verse afectado por el error.
  • Genericidad del código: "Cuanto más genéricos son los programas, más reusable es el código pero también es más complejo y difícil de mantener." Esto significa que cuando más genérico es, por ende es más reducido, y llega un punto en el cual el código es tan breve y tan genérico que se hace ilegible.

Lenguajes de programación

  • Lenguaje compilado vs interpretado: la diferencia es que un lenguaje compilado requiere de una compilación antes de ser ejecutado, que lo convierte en lenguaje máquina. En cambio, el lenguaje interpretado convierte el código a lenguaje máquina al mismo tiempo que se ejecuta.
  • Diferencias entre un compilador, un intérprete y un transpilador: un compilador genera un programa que puede funcionar por sí mismo, un intérprete traduce un programa para que pueda ser ejecutado por la máquina y el transpilador crea un programa en un lenguaje que necesita un intérprete para poder ser ejecutado.
  • Lenguajes débilmente tipados vs fuertemente tipados: en los lenguajes débilmente tipados no es necesario declarar el tipo de datos con el que trabajamos y éste se puede alterar en cualquier momento, aunque a veces pueda dar resultados inesperados. Sin embargo, en los fuertemente tipados tenemos que declarar el tipo de datos que estamos utilizando en cada momento y no se pueden alterar, al igual que tampoco se puede operar entre datos de distinto tipo.
  • Diferencias entre tipado estático y dinámico: en el tipado estático tenemos que declarar el tipo de datos que utilizamos en cada momento, sin embargo, en el dinámico no es necesario. Las diferencias son en la velocidad de carga, pues el estático carga más rápido ya que los datos están ya tipados y no hay que tiparlos al tiempo de ejecución, como en el dinámico. Otro error común es el resultado de comparar dos tipos de datos distintos, en el estático no tendrás ese problema porque no te permitirá ejecutar el código, pero en el dinámico es necesario manejarlo.
  • Timetomarket: es el tiempo que se tarda desde que se empiezan a crear las funcionalidades del lenguaje hasta que está finalizado y operativo.
  • Sobrecarga de operadores: es la reutilización del mismo operador para distintas operaciones distintas, dependiendo de sobre qué opera. +info
  • Interpolación de cadenas: es el proceso de cambiar una cadena literal por su valor correspondiente en la ejecución.

Bases de datos

  • Diferencias entre bases de datos, SGBD y SGBDR:en una base de datos, los datos son independientes de la aplicación, por tanto, pueden ser usados por cualquiera de ellas y se visualiza normalmente como una hoja de cálculo. En un SGBD, se ha de configurar qué tipo de datos hay, la estructura, las restricciones y controlar la concurrencia, además de que esos datos pueden manipularse con consultas, actualizaciones... Por último, los SGBDR son muy parecidos a los SGBD, pero éstos últimos son para bases de datos relacionales.
  • Sistemas gestores de datos más importantes del mercado: +info
    • Microsoft Access (relacional)
    • MySQL (relacional, libre)
    • Oracle Database (relacional)
    • MariaDB (relacional, libre)
    • MongoDB (no relacional, orientado a documentos, libre)
    • IBM Informix (relacional)
  • APIs abiertas:
    • Firebase: es una plataforma en la nube para el desarrollo de aplicaciones web y móvil. +info
    • GraphQL: es una herramienta alternativa a REST que permite comunicar clientes y servidores a través de consultas.
  • SGBD NoSQL:
    • Apache Cassandra: base de datos NoSQL basada en el modelo de almacenamiento clave-valor. +info
    • MongoDB: sistema de base de datos NoSQL orientado a documentos.
    • Redis: es un motor de bases de datos en memoria. +info
  • Campos autonuméricos: tipo de datos que numera de forma correlativa y automática los nuevos registros en una base de datos.
  • Claves: campo que identifica de forma única cada fila de una tabla de base de datos.
  • Claves ajenas: campo de una tabla que se corresponde con la clave primaria de otra tabla y las relaciona.
  • Vistas: es una tabla cuyo contenido está definido por una consulta.
  • Diagramas entidad-relación: diagrama que muestra como las entidades se relacionan entre sí en una base de datos.
  • Formas normales:
    • 1ª forma normal: todos los atributos son átomicos, la tabla contiene una clave primaria única y no puede ser nula, no debe existir variación en el número de columnas, los campos no clave deben identificarse por la clave, debe existir independencia en el orden tanto de filas como de columnas.
    • 2ª forma normal: se cumple lo mismo que en la 1ª además de que los atributos no clave dependen de la clave principal.
    • 3ª forma normal: se cumple lo mismo que en la 2ª además de que no debe existir ninguna dependencia transitiva entre los atributos que no son clave.
  • Lenguajes DDL y DML: las sentencias DDL son para crear bases de datos y sus componentes. Las sentencias DML son para insertar, modificar, borrar y consultar datos en la base de datos.
  • Lenguaje de consultas SQL:
    • Inserciones: instrucción para añadir registros en una tabla de base de datos (INSERT).
    • Borrados: instrucción para borrar registros en una base de datos (DELETE).
    • Actualizaciones: instrucción para actualizar la información de una tabla en base de datos (UPDATE).
    • Consultas: instrucción para consultar los datos de una base de datos (SELECT).
    • Alias: instrucción para renombrar columnas en las consultas de una base de datos (AS).
    • Condiciones WHERE: instrucción para filtrar las consultas en una base de datos (WHERE).
    • Consultas agrupadas: instrucción que considera registros con campos de igual valor y los procesa de la misma manera.
  • Resultset: conjunto de filas obtenidas desde una base de datos, así como metadatos sobre la consulta y tamaños de cada columna.
  • Transacciones: secuencias de trabajo realizadas de forma ordenada y separada en una base de datos.
  • Prepared statements: característica utilizada para ejecutar la misma o similar declaración en una base de datos repetidamente con alta eficiencia.
  • Inyección de código: infiltración de código intruso que se aprovecha de una vulnerabilidad en la base de datos de una aplicación para poder operar con ella.
  • GUI para mysql:
    • phpMyAdmin: herramienta escrita en PHP para manejar MySQL a través de páginas web utilizando el navegador.
    • MySQL Workbench: herramienta visual de diseño, administración, gestión y mantenimiento de bases de datos MySQL.
    • HeidiSQL: software de código abierto que permite conectarse a servidores MySQL.
    • TOAD: aplicación informática de desarrollo y administración de bases de datos SQL.

Control de versiones

  • El control de versiones es la gestión de cambios que se realizan a lo largo del tiempo sobre un proyecto.
  • Ventajas que aportan los controladores de versiones:
    • Trazabilidad: es un seguimiento del proyecto desde su estado original hasta su estado final.
    • Historial de cambios: es un historial en el que se registra cada cambio realizado en el proyecto. Además de quién hizo ese cambio y en qué momento.
    • Reversión: es la posibilidad de poder volver a un estado anterior del proyecto si los cambios que se hicieron a posteriori dieron problemas.
    • Trabajo en grupo: el control de versiones permite el trabajo en grupo ya que puede conectarse al proyecto cualquier persona en cualquier lugar que tenga acceso a la red y con las características descritas previamente, que hacen más sencillo y fiable el desarrollo en equipo de un proyecto.
    • Trabajo en ramas: es una posibilidad que nos da el control de versiones de modificar copias del proyecto aisladas de éste para que los cambios no afecten directamente al proyecto principal. Además, se puede hacer un control de permisos en cada rama para dar seguridad al trabajo.
  • Herramientas:
    • cvs: aplicación informática que implementa un sistema de control de versiones, mantiene todo el registro de trabajos y cambios y permite que distintos desarrolladores colaboren.
    • subversion svn: software de gestión de versiones que pertenece a Apache. Destaca por su fácil implementación en un servidor Apache y su bajo coste a la hora de crear ramas.
    • git: software de gestión de versiones creado desde sus inicios para Linux pero después se lanzó para todo tipo de proyectos. Permite la creación de un proyecto nuevo a partir de una ramificación y una gestión rápida y sencilla de las versiones de un proyecto.
    • mercurial: software de gestión de versiones con gran rendimiento y escalabilidad. Tiene una interfaz y un matenimiento muy sencillos.
  • Repositorios git:
    • GitHub: es un sistema de gestión de proyectos que utiliza el sistema de control de versiones Git.
    • Bitbucket: es un servicio de alojamiento web para proyectos que utilizan un control de versiones basado en Mercurial o Git.
    • Gitlab: es un servicio web de control de versiones basado en Git que además ofrece alojamiento de wikis y seguimiento de errores.
