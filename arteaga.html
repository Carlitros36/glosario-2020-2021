Protocolo HTTP
Es un protocolo de comunicación que permite la conexión en la World Wide Web.
El proceso de carga de una página web
El ordenador manda la solicitud al router para que este lo envie hacia el exterior llegando al servidor que tiene la informacion y luego se hace la tranferencia de datos entre el servidor y el ruter mediante el protocolo HTTP o HTTPS, cuando la información llega al router este lo envia al ordenador que lo solicitó donde son analizados y mostrados por un navegador web
TCP
Se encarga que los ordenadores se comuniquen de una forma sencilla y transmitan información a travez de la red
IP
Son unos números que identifican a un dispositivo para que se conecte a la red
URL
Localizador uniforme de recurso y se refiere a la dirección única que identifica a una página web en Internet
Aplicaciones cliente-servidor vs P2P
En el cliente-servidor senecesita un servidor y unos clientes para que funcione, mientras para el P2P cualquier usuario puede ser el cliente o el servidor
¿Qué es una cookie y para qué sirve? ¿cómo podemos establecer una sesión usando cookies? ¿donde viaja la cookie dentro del paquete HTTP?
Las cookies son necesarias porque el protocolo HTTP que se emplea en la Web para transmitir las páginas web, es un protocolo sin estado, y no ofrece un mecanismo para mantener el estado, es decir, la historia de las peticiones y acciones realizadas por un usuario entre diferentes peticiones. Es una pequeña información enviada por un sitio web y almacenada en el navegador del usuario, sirve para recordar accesos y conocer los hábitos de navegación
¿Qué son los tipos MIME y para qué sirven?
Son la manera standard de mandar contenido a través de la red, adjunta cada archivo en la cabecera

Metodologías del desarrollo del software:
  • Modelo en cascada: representa una unidad de desarrollo con un pequeño descanso en el medio. Por lo tanto, cada siguiente etapa inicia tan pronto como la anterior haya culminado
  • Modelo en espiral: refleja la relación de tareas con prototipos rápidos, mayor paralelismo y concurrencia en las actividades de diseño y construcción
  • Modelos basados en prototipos: es un procedimiento de desarrollo especializado que permite a los desarrolladores la posibilidad de poder solo hacer la muestra de la resolución para poder validar su esencia funcional ante los clientes, y hacer los cambios que sean fundamentales antes de crear la solución final auténtica
  • Extreme programming: se utiliza principalmente para evitar el desarrollo de funciones que actualmente no se necesitan, pero sobre todo para para atender proyectos complicados
Integración contínua (concepto, servidores)
Los desarrolladores integran su código una o varias veces al día para que los demas de su equipo tengan acceso y puedan corregir los fallos ráidamente, TDD es un desarrollo guiado por pruebas de software

Evaluación de la calidad del software (concepto, herramientas disponibles)
  • Funcionalidad. Las funciones del software son aquellas que buscan satisfacer las necesidades del usuario
  • Confiabilidad. La capacidad del software de mantener su rendimiento bajo ciertas condiciones durante cierto período de tiempo
  • Usabilidad. Basada en el esfuerzo necesario para utilizar el software por parte de un grupo de usuario
  • Eficiencia. Basada en la relación entre el nivel de rendimiento del software y el volumen de recursos utilizado, bajo ciertas condiciones.
  • Capacidad de mantenimiento. Basada en el esfuerzo necesario para realizar modificaciones específicas
  • Portabilidad. Basada en la capacidad del software para ser transferido de un entorno a otro

Procesamiento asíncrono, Callbacks, callback hell, promesas y programación reactiva
Un cuando un proceso se ejecuta en segundo plano y que el usuario no tenga que esperar a que termine. Los callbacks es solo el nombre que se usa para referirse a ciertas funciones de JavaScript, En lugar de devolver inmediatamente un resultado como la mayoría de las funciones, las funciones que utilizan callbacks toman un tiempo para producir un resultado, la causa del callback hell es cuando las personas intentan escribir JavaScript de forma tal que la ejecución se realiza visualmente de arriba a abajo, una promesa es un objeto devuelto al cuál se adjuntan funciones callback, en lugar de pasar callbacks a una función

Paradigmas de la programación
  • ¿En qué consiste paradigma declarativo frente al paradigma imperativo?
    • Paradiga imperativo son los programas consisten en una sucesión de instrucciones o conjunto de sentencias, como si el programador diera órdenes concretas. En el paradigma imperativo utiliza el principio del razonamiento lógico para responder a las preguntas o cuestiones consultadas

    • La programación funcional es un paradigma de programación declarativa basado en el uso de verdaderas funciones matemáticas
Técnicas de programación imperativa
  • Programación estructurada
    • Es un tipo de programación imperativa donde el flujo de control se define mediante bucles anidados, condicionales y subrutinas
  • Programación modular
    • Consiste en dividir un programa en módulos o subprogramas con el fin de hacerlo más manejable y legible
  • Programación orientada a objetos
    • Se construyen modelos de objetos que representan elementos (objetos) del problema a resolver, que tienen características y funciones. Permite separar los diferentes componentes de un programa, simplificando así su creación, depuración y posteriores mejoras

    • No hay clases en JavaScript y la herencia en JavaScript es herencia entre objetos
  • Explica en qué consisten las Interfaces fluidas

Lenguajes débilmente tipados vs lenguajes fuertemente tipados: diferencias, ventajas y desventajas
El tipado re refiere a como declaramos las variables, ya que en algunos lenguajes necesitamos declararlas y en otras no.
Tipado Ventajas Desventajas
Débil
  • Nos olvidamos de declarar el tipo
  • Podemos cambiar el tipo de la variable sobre la marcha
  • Escribimos menos código
  • Al hacer operaciones, a veces éstas salen mal
  • Puede dar errores al comparar números que creemos que son números, pero no lo son
  • Hay que castear muchas veces
  • Código menos expresivo
Fuerte
  • Código expresivo
  • Menos errores
  • Escribimos mas código
¿Qué diferencia existe entre un tipado estático y uno dinámico?
El tipado estático nos obliga a definir desde el principio el tipo de una variable, el tipado dinamico nos da la facilidad de no definir los tipos al declarar una variable
¿Qué es el timetomarket de un lenguaje?
Es el tiempo comprendido desde que un producto o servicio es concebido hasta que está disponible para el usuario final
¿En qué consiste el duck typing?
La idea es que no necesita un tipo para invocar un método existente en un objeto; si se define un método en él, puede invocarlo.
¿Qué es la sobrecarga de operadores?
Cuando reutilizando el mismo operador con un número de usos diferentes, y el compilador decide como usar ese operador dependiendo sobre qué opera
¿Qué es la interpolación de cadenas?
Consiste en introducir valores de variables o expresiones en el interior de cadenas de texto sin usar los tradicionales mecanismos, molestos y propensos a errores, como la concatenación

Base de datos

Diferencia entre base de datos, SGBD y SGBDR
Un SGBD (sistema de gestion de base de datos) es un programa informatico que permite crear, modificar y mantener una base de datos. Un SGBDR (sistema de gestion de base de datos relacionales) es aquel que sigue el modelo relacional.
Sistemas gestores de bases de datos más importantes del mercado
Microsoft Access (relacional) Microsoft SQL Server (relacional) MySQL (relacional) Oracle Database (relacional) OrientDB (orientado a documentos) CouchDB (orientado a documentos) Db2 de IBM (relacional) IMS de IBM (jerárquico) IBM Informix (relacional) MariaDB (relacional) Sybase ASE (relacional) MongoDB (orientado a documentos) PostgreSQL (combina relacional y orientado a objetos) Firebird (relacional) Caché (jerárquico)
APIs abiertas
Muchas empresas ofrecen sus APIs de forma abierta para que los desarrolladores puedan acceder de forma libre, firebase es una plataforma desarrollada por Google para el desarrollo de aplicaciones web y móviles
SGBD NoSQL
NoSQL es una amplia clase de sistemas de gestión de bases de datos que difieren del modelo clásico de SGBDR en aspectos importantes, siendo el más destacado que no usan SQL como lenguaje principal de consultas. Los datos almacenados no requieren estructuras fijas como tablas, normalmente no soportan operaciones JOIN, ni garantizan completamente ACID y habitualmente escalan bien horizontalmente
Concepto de campos autonuméricos. Claves. Claves ajenas
Son los datos con los cuales relacionaremos las tablas de una base de datos que van aumentando automaticamente a medida que se introducen datos a la base de datos
Concepto de vista
Es una consulta que se muestra como una tabla a partir de un conjunto de tablas
Diagramas de entidad-relación para modelar una base de datos relacional
Un modelo entidad-relación es una herramienta para el modelo de datos, la cual facilita la representación de entidades de una base de datos
Formas normales, hasta la tercera forma normal
Son reglas que se aplican a las relaciones obtenidas al modelo relacional, en la primera forma se elimina los valores repetidos en la base de datos, en la segunda forma se encarga que ningun campo este fuera de su lugar, en la tercera forma eliminar los atributos no claves que no dependen de la clave primaria, sino de otro atributo
Lenguajes DDL y DML
DML es un lenguaje de manipulación de datos, en sql los mas comunes son SELECT, INCERT, DELETE, UPDATE. DDL es un lenguaje de definicion de datos es decir como tienen que residir los datos en la base de datos como por ejemplo el CREATE, ALTER, DROP, RENAME

Repositorios git
Un repositorio de Git es un almacenamiento virtual de tu proyecto que te permite guardar versiones del código a las que puedes acceder cuando lo necesites

Integración continua

Pruebas unitarias
La prueba de uno de los modulos de un programa, Junit es una librería desarrollada para poder probar el funcionamiento de las clases y métodos que componen nuestra aplicación, y asegurarnos de que se comportan como deben ante distintas situaciones de entrada
Pruebas de integracion
Servidores: Jenkins, Travis CI, TeamCity, CircleCI
Jenkins ayuda en la automatización de parte del proceso de desarrollo de software mediante integración continua y facilita ciertos aspectos de la entrega continua
Issue tracking: Jira, backlog, trac. Redmine, mantis
Jira es una herramienta en línea para la administración de tareas de un proyecto, el seguimiento de errores e incidencias y para la gestión operativa de proyectos
Tecnologías para la inspección y la mejora del código: codeclimate, sonarqube
Code Climate analiza su código en busca de complejidad y duplicación para determinar cambios en la calidad y los puntos críticos de endeudamiento técnico de la superficie, SonarQube proporciona una descripción general del estado general de su código fuente y, lo que es más importante, destaca los problemas encontrados en el nuevo código.

Despliegue

Microservicios y Message Brokers: RabbitMQ, Kafka, Redis, SQS
Message Brokers es un programa intermediario que traduce los mensajes de un sistema desde un lenguaje a otro, a través de un medio de telecomunicaciones
Monitorización: new relic, datadog, dynatrace
Datadog es un servicio de monitorización para aplicaciones en la nube, que proporciona monitorización de servidores, bases de datos, herramientas y servicios, a través de una plataforma de análisis de datos basada en "Software como Servicio"
Infraestructura
La infraestructura como servicio (IaaS) es una infraestructura informática inmediata que se aprovisiona y administra a través de Internet. Es uno de los cuatro de servicios en la nube, junto el con software como servicio (SaaS), la plataforma como servicio (PaaS) y la tecnología sin servidor
Contenedores
Docker es una plataforma de software que le permite crear, probar e implementar aplicaciones rápidamente. Docker empaqueta software en unidades estandarizadas llamadas contenedores que incluyen todo lo necesario para que el software se ejecute, incluidas bibliotecas, herramientas de sistema, código y tiempo de ejecución. Con Docker, puede implementar y ajustar la escala de aplicaciones rápidamente en cualquier entorno con la certeza de saber que su código se ejecutará
